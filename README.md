## 命名規則
1. バージョン名
   1. マスターバージョンは0から始める。最初のリリースが1である。
   2. 以下`.`で二桁ごとに区切る
   3. (Master).(機能バージョン).(ブランチバージョン)とする
   4. リリース直後は1.00.00である
2. ブランチ名
   1. 操作する機能、ファイル名、関数名のいずれかを冠する
   2. READMEは1.0より始める
   3. その他は(機能名)(Master).(操作する機能)
   4. 訂正するときは`fix:`をPRに冠するようにすること
   5. なんとなく雰囲気で
3. ファイル名
   1. 日本語は使わない
   2. 適当に
4. 関数名
   1. 日本語は使わない
   2. 適当に
5. 変数名
   1. 日本語は使わない
   2. 適当に
